# Session: ascii-renderer-poc

## Goal
Build a nondeterministic ASCII art renderer that takes YAML layout specs from an LLM and produces unique terminal art every run. Started as /map command evolution, became a standalone Rust project.

## Current State
- **Repo**: ~/projects/ascii-renderer (cargo init, not yet git-pushed)
- **Deps**: rand, console_engine (unused so far)
- **Working primitives**: Truchet fill, DLA crystal growth (3-zone glyph bands), binary tree growth (GRIS-style box-drawing), flower/rosette stamps (5 styles)
- **Seeded RNG**: deterministic output per seed, different art per seed
- **Plugin repo**: ~/projects/claude-research -- has /map and /vibes commands, secret scanning hooks, save/load commands all working

## Problem/Context
- LLMs can't count characters, so ASCII art alignment always drifts
- Needed a renderer that handles all spatial math while LLM declares intent
- Each run should look different -- same spec, different visual
- Art references: Armored Core 3, Ghost in the Shell, Evangelion NERV, GRIS trees, Amsterdam wrought iron

## Solution/Approach
- Rust CLI at ~/projects/ascii-renderer
- 4 primitives implemented as POC
- Planned algorithms: WFC, Wolfram CA, L-systems, Perlin noise, Truchet tiles, DLA, Markov chains, SDF, space-filling curves
- Planned crates: console_engine (canvas), wfc (gridbugs), fast_hilbert, noise
- YAML spec format designed but not yet implemented
- Three rendering zones for DLA: border (structural ╍┅┉), mid (geometric ⟋⟍⧸⧄⧈), far (braille corruption ⡷⣟⢿⣻)

## Key Insights
- Truchet tiles: 2 chars, coin flip, maximum emergent complexity from minimum info
- DLA: spawn particles near cluster (not edges) for denser growth, 3000 particles + 2000 walk steps for good coverage
- set -euo pipefail + ls on empty glob = silent crash, use nullglob+array
- Round glyphs (○●◆) clash with angular Truchet (╱╲), angular crystals (╳╬┼╪) blend better
- Braille characters make excellent corruption/noise zone glyphs
- GRIS trees: binary splits with ╭─┼─╮, all tips at same height, strict grid geometry
