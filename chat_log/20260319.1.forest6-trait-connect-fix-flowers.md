## Key Insights
- connect_glyph confused "arriving from" with "traveling in direction". Every turn char was swapped within its pair. Fix propagates to ALL pen trees in the codebase.
- Trees don't care about overwrite rules. They draw cells. Layering is the compositor's problem. Removed can_overwrite from trait.
- should_branch needs integer index+count, not float fractions. Float quantization on a discrete grid produces gaps. ASCII = integers.
- Randomness from outside: tree is pure function of (TreeParams + rng state). Tree consumes entropy, doesn't own it.
- Spiral uses default grow() loop. Split and candelabra override it entirely.
- Flowers should NOT use TreePen. Box-drawing chars = tree branches. Flowers use radial Unicode glyph placement.
- Stem and bloom grow on different energy curves (stem linear, bloom accelerates 0.4-0.7).
- Terminal chars ~2:1 height:width, need 1.8x correction on x for radial placement.
