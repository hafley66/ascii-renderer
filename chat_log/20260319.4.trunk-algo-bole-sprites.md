### Bole design evolution:
1. Started with geometric bar stamps -- too symmetric, disconnected
2. Moved to TreePen walks -- too scattered, not coherent shapes
3. Landed on sprite-style patterns (like draw_flower) with coherent char families per style
4. Added asymmetry via independent lw/rw widths from rng
5. Squashed Frame from 5 rows to 2 (landscape rectangle, not square)
6. Each style uses a distinct Unicode char family: crescents, braille, box-drawing, diamonds, chevrons

## Key Insights
- Haiku agents work well for focused extraction tasks (TrunkAlgo wiring) but struggle with aesthetic design (bole visual quality)
- User wants "autistic delegation" -- send haiku agents for code tasks, conserve main context tokens
- Boles are sprite-art, not path-walks. They should be compact coherent shapes using unified char families, like draw_flower patterns but horizontal
- Boles must: be wider than tall (landscape), connect to trunk via valid junction chars, touch ground line, have asymmetry
- The user explicitly dislikes: symmetric shapes, disconnected layers, square/vertical proportions, generic box-drawing corner walks
- Frame style works best as a 2-row squat rectangle with a "behind" frame offset and interior fill
- User called the bole "the thick boy root that should be horizontal" -- NOT underground root systems, NOT vertical trunk
