Continue building the ascii-renderer: add more tree/fruit primitives, fix Aztec diamond algorithm, build a layout engine, and add markdown input parsing.

## What Was Built This Session

1. **Three new tree primitives**: pine (layered triangular tiers with needle fill), willow (binary splits with drooping branch tips using dashed verticals), palm (tall trunk with radiating fronds and coconuts)
2. **Five fruit stamps**: apple, cherry pair, citrus, berry cluster, pear
3. **Two new demo modes**: `forest` (mixed scene with all tree types), `fruits` (showcase)
4. **CLI help text**: `--help` flag with all modes, themes grouped by category, usage examples
5. **Aztec diamond fix**: corrected domino shuffling algorithm. Bugs were: DELETE happened after SLIDE (should be before), N-S collision pattern was inverted (checking N,N,S,S instead of S,S,N,N), CREATE fill pattern was inverted. Also switched rendering from half-block glyphs to 2-char-wide background-colored cells for aspect ratio correction.
6. **Layout engine**: ContentBlock/ContentItem types, wrap_text (greedy line breaking), measure_block, render_block. Two layout strategies: `layout_two_col` (fixed two-column) and `layout_bsp` (BSP randomized partitioning with min size constraints and aspect ratio bias)
7. **BSP layout**: recursive binary space partition with min_w/min_h constraints, aspect-ratio-biased split axis, largest-area-first block assignment, empty leaves filled with procedural primitives (trees in large regions, frets/aztec diamonds in medium, flowers/fruits in small)
8. **Markdown parser**: pulldown-cmark integration. Headings group with following content into single ContentBlock (fixed grouping bug where titles scattered away from content). Supports paragraphs, lists (bulleted), blockquotes, code blocks, horizontal rules. H1 gets bracket treatment.
9. **Pattern-decorated md mode**: empty BSP leaves get trees, frets, aztec diamonds, flowers, fruits based on region size. Fret borders on all 4 canvas edges. Content margin accounts for border band depth.

## Key Insights
- Aztec diamond domino shuffling order: DELETE (facing-toward pairs on old grid) -> SLIDE (embed + directional shift into new grid) -> CREATE (facing-apart pairs in empty 2x2 blocks). Parity flips each iteration.
- In r-down coordinates: "bad" horizontal pair = top S,S bottom N,N (facing toward). Fill pattern = top N,N bottom S,S (facing apart).
- BSP is the right layout primitive for constrained randomization in terminal grids: guaranteed no overlap, seed-deterministic, produces natural visual variety
- Markdown sections should group heading + following content into one block, not separate blocks. Flush happens when the NEXT heading starts, not when the current heading ends.
- Terminal aspect ratio (~1:2) means Aztec diamond needs 2 chars per cell horizontally
