## Goal
Make BoleStyle energy-aware, wire boles into TreeDrawer grow() pipeline, create multiple bole test modes (boles1/2/3), iterate on visual quality of each style.

## Current State

### What got built this session:

- **grow() pipeline**: bole draws first, trunk starts from bole's returned coords, no blind │ stomp
- **pen.last_dir = Some(MoveDir::Up)** at start so first trunk step connects properly

- **boles3 mode**: 6 refined styles (styles 12-17) -- the current iteration target

## Key Insights
- Smooth bell/sine curves make mounds, not boles. Boles need SHELF structure: wide flat base then sharp drop
- Braille chars need structural differentiation not just density gradients -- shelf vs mound is the key
- Frame works because overlapping rectangles with junction glyphs create depth illusion
- Chevron works because off-center drift creates emergent diamond patterns
- Croissant failed 4+ times across haiku/opus because crescent Unicode chars (◠◡◜◝◟◞) don't connect well enough to form coherent shapes algorithmically
- User wants haiku agents for code tasks but they need VERY precise instructions -- creative freedom produces bad ASCII art
- Don't let haiku see previous bad implementations -- it gets tainted
- User explicitly said: never touch Frame (14) or Chevron (16) again
- Parallel agents on same file cause git stash conflicts -- use worktree isolation next time
- lighten/darken compound too aggressively -- darken(darken(color, 15), 30) = invisible on dark backgrounds
- User prefers numbered boles modes (boles1/2/3) for iteration but wants descriptive style names within each mode
