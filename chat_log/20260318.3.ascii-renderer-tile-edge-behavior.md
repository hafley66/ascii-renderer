# Session: ascii-renderer-tile-edge-behavior

## Goal
Implement TileEdgeBehavior: per-variant bespoke edge control for the tile skew system, so patterns like Seigaiha close their arches, GrannySquare closes boxes, Nowaki extends strokes, etc. instead of generic dropout at boundaries.

## Current State
- TileEdgeContext struct + edge_behavior() free function implemented in src/fills.rs
- All 10 tile variants handled: mirror_edge (Seigaiha/ShellStitch/Shippo/BishamonKikko/CrocodileScale), granny_edge (GrannySquare), nowaki_edge (Nowaki), default_edge (Asanoha/Yabane/Higaki)
- fill_tile_ex rewired: signed distances, per-variant mirroring before tile.at(), edge_behavior dispatch for outside/near-edge cells
- 16 tests passing
- Committed as c8719e0

## Design decisions made
- Free function (edge_behavior dispatching on TileVariant enum) instead of trait/dynamic dispatch -- matches existing pattern in fills.rs (make_tile, render_fill)
- Mirroring still happens in fill_tile_ex before tile.at() since edge functions don't hold TilePattern ref; edge_behavior controls draw-vs-skip and can emit arbitrary chars (GrannySquare does)
- Signed edge distances: negative = inside by N cells, positive = outside
- near_edge trigger: within one tile period of bottom/right edge AND skew > 0
