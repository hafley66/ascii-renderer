## Key Insights
- Three parallel fill dispatch systems (LeafFill, FlowFill, inline biome match arms) all resolved to the same ~9 terminal fill functions -- collapsed into one FillGen + render_fill
- fills.rs and sprites.rs are independent (neither imports the other), so scene.rs sits above both: types -> color -> fills -> sprites -> scene -> walker -> biomes
- Layer carries its own palette because terrain uses 4 different palette rotations across layers (mtn_palette, hill_palette, gnd_palette, etc.)
- Contour ridges, tree scatter, moon are imperative post-passes that don't fit fill+mask model -- stay as procedural code after render_scene()
- User wants multi-pass compositing where biomes are layer passes gated by masks, not region owners in strip partitions
- Weave fill always uses lighten(color, 30) for color2 -- baked into render_fill
