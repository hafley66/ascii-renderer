# Session: ascii-renderer-tile-noise-system

## Goal
Build a parameterized tile pattern and noise fill system for the ascii-renderer, enabling programmatic generation of sashiko, crochet, and other decorative patterns with controllable randomness.

## Current State
- 10 tile patterns implemented: asanoha, seigaiha, shippo, bishamon kikko, yabane, nowaki, higaki, shell stitch, granny square, crocodile scale
- 6 noise variants implemented: truchet, higaki (coherent runs), higaki-static (per-cell random), grass, static, dot
- TilePattern struct with period_x/period_y, row_offset, stagger_rhythm
- TileParams struct with variant, density, stagger_override, rhythm_override + randomized() constructor
- Noise system has coherence parameter controlling run length (0.0 = independent, 0.7 = ~3.3 cell runs)
- fill_tile_pure() for deterministic baseline, fill_tile_ex() for parameterized fills
- All wired into LeafFill enum and LeafWalker (both organic and geometric moods)
- New modes: `tiles` (pure deterministic), `tiles-rand` (randomized params), `noise` (all noise variants)
- fill_truchet() extracted to replace 6+ inline duplications (only applied to default mode so far)
- All 10 tests passing

## Key Architecture
- TilePattern: small 2D char array + stagger params, `at(x,y)` does periodic lookup
- NoiseVariant: weighted glyph table + coherence, Markov-chain-style run generation
- LeafFill::Tile(TileParams) and LeafFill::Noise(NoiseVariant) added to walker
- Walker energy maps to jitter: high energy = clean, low energy = weathered

## Key Insights
- User wants pure deterministic baseline as the default view, randomized as a separate mode
- Coherence on noise fills creates "long runs with interruptions" vs pure per-cell random -- both should be available (Higaki vs HigakiStatic)
- Stagger rhythm (how many tile-rows before offset shifts) is a distinct dimension from stagger amount
- Phase shift per-instance is cheap high-impact variation
- User explicitly noted: "deterministic rand" needed in next pass -- seed-controlled randomization
