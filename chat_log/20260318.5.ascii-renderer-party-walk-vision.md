# Session: ascii-renderer-party-walk-vision

## Goal
Redesign the stem/path walker into a rich generative world-walk system where each node is a "party" with character, and the global layout has a personality (taper, branching, color arc).

## Current State
- Shape-aware skew shipped: `fill_tile_ex` accepts `skew_boundary: Option<&dyn Fn(usize,usize)->f32>`, `fill_masked` passes mask fn automatically
- Stem walker shipped: `path_walk_stem` generates sinuous spine, arc-length resamples to 5-8 leaf nodes, each leaf gets shape based on stalk position (trapezoid/parallelogram/triangle), `draw_stalk` renders spine with directional chars
- `stem` mode added to main.rs
- Criticism: stem is too structurally uniform -- same shape thresholds, same size lerp, no per-plant character

## Problem/Context
The stem walker needs to become a much richer system. Nodes should be "parties with character" not just tile fills. The global layout should have a drawn personality. The full vision is large.

## Solution/Approach

### Two node modes (top-level picker per node):
**Mode A — Landscape (~40%)**
- Pass 1: Background (sky, starfield, rain, dot noise) masked to node shape
- Pass 2: Ground band (bottom 25-40% of node) — grass, cement, water
- Pass 3: Foreground elements — tree, mountain peak, pillar, floaty guys, flowers

**Mode B — Composition (~60%)**
- CenterpieceWithSurround: central face/explosion/mask/aztec + surrounding tile patterns + diamond/circle/hex container
- Cluster(arrangement, n, relationship): N related patterns in hex/ring/spiral/loose/grid arrangement
- "Related" = same tile family, color family, or size family — constraint that gives cluster coherence

### Global layout personality (drawn once per plant):
```
TaperDir: Uniform | Left | Right | Up | Down | Center | Radial
branch_factor: 0.1..2.0   (straight vs wild walk)
taper_strength: 0.0..1.0
landscape_bias: f32       (weight toward Landscape vs Composition)
```

### Additional systems identified (all confirmed):
- **Path character**: connective tissue between nodes — vine, thick border, river, wall, ruled ornament line
- **Global atmosphere overlay**: rain/snow/static/fog as final pass over entire scene
- **Color arc along walk**: palette shifts warm→cool or light→dark as walk progresses
- **Negative space nodes**: intentionally empty/sparse nodes for rhythm
- **Nesting/recursion**: a node can contain a miniature version of the whole system
- **Symmetry character draw**: whole plant mirrored, or cluster radially symmetric (mandala)
- **Text/glyph nodes**: pure decorated typography — runes, symbols, single large kanji with pattern fill
- **Format ambition**: eventually renders to terminal, file, generative feed

### Floaty guys:
Small autonomous things scattered or in 1D rows:
- CA snapshot (small isolated cellular automata pattern)
- Flower sprite (already exists)
- Explosion/fireworks (radiating pattern from center)
- "thing × N in a row" (1D repetition of a small pattern)

### Sky and Rain:
- Sky: sparse cloud/dot pattern, horizontal drift feel — background layer fill
- Rain: dense diagonal ╱ strokes with · drops — can be per-node or global atmosphere overlay

## Key Insights
- `render_scene` already composites Vec<Layer> bottom-to-top — each node just produces Vec<Layer> and they merge into global scene. No structural change needed for multi-pass.
- "Related" constraint on clusters is what separates them from random scatter
- Taper/layout personality is orthogonal to node content — any node type can appear at any size in any taper config
- Hex shape mask doesn't exist yet (circle/ellipse approximation could work short-term)
- The path connective tissue should be as designed as the nodes — currently just dotted trail
- Negative space is a design tool, not an absence of design

## Open Questions
- For Cluster: self-contained (packs N things internally) or uses walk/taper layout system to position sub-elements?
- Hexagon: new mask_hexagon() function, or approximate with existing shapes?
- Color arc: interpolate between two drawn palette endpoints, or cycle through named themes?
- Path character: separate draw pass after render_scene, or encoded as thin layers between node layers?
- At what point does "stem" mode become "party" or "world-walk" mode?
